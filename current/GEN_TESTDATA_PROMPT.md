# Промт: генерация тестовых данных по цементированию скважин

Один и тот же текст выдаётся каждому генератору, **меняются только две
строки в разделе «Твоя квота»** — диапазон номеров скважин и имя файла.
Остальное обязано совпадать дословно, иначе данные разойдутся между
партиями.

Загружать порядок: сначала `current/testdata_dictionaries.json` (один
раз, вручную), потом партии скважин. Всегда через `--dry-run` первым
прогоном.

---

## ТЕКСТ ПРОМТА (копировать целиком, ниже разделителя)

---

Ты генерируешь тестовые данные для системы учёта цементирования нефтяных
скважин. Данные пойдут в реальную базу через загрузчик, который проверяет
каждое поле и отвергает запись целиком при любом несовпадении. Поэтому
точность формата важнее красоты содержания.

### Что нужно на выходе

Один JSON-файл. **Только JSON, без пояснений, без markdown-ограждения,
без строк вида ```json.** Первый символ ответа — `{`, последний — `}`.

Кодировка UTF-8, русский текст как есть (не `\u0410...`).

### Твоя квота

* Количество скважин: **10**
* Номера скважин (`int_number`): **от 101 до 110** ← *(вторая партия: 201–210, третья: 301–310)*
* Имя файла в ответе не нужно, просто JSON.

Номера не должны пересекаться с чужими партиями — это единственное, что
отличает твою порцию от соседней.

### Структура файла

```json
{
  "well": [
    { ...поля скважины..., "cementing_stage": [ ... ], "productive_horizon": [ ... ] },
    { ...вторая скважина... }
  ]
}
```

Вложенность выражается **массивом под именем дочерней таблицы внутри
объекта родителя**. Дерево целиком:

```
well
  cementing_stage                     (ступень цементирования)
    cementing_interval                (интервал)
      cementing_buffer                (буфер)
      material_test                   (лабораторное испытание)
      akc_by_material                 (АКЦ по материалу)
    cementing_reper                   (репер)
      cementing_reper_component       (компонент репера)
  productive_horizon                  (продуктивный горизонт)
    akc_by_horizon                    (АКЦ по пласту)
```

### Железные правила формата

1. **Все значения — строки.** `"3000"`, а не `3000`. `"1.92"`, а не
   `1.92`. Дробный разделитель — **точка**, не запятая.
2. **Никаких служебных полей.** Не указывай `id`, `dep_*`, `rel_main`,
   `active`. Связь родитель–ребёнок выражается только вложенностью.
   Любое служебное поле в файле → отказ записи.
3. **Никаких вычисляемых полей.** `calc_volume_deviation` у интервала
   считается системой. Указывать его нельзя.
4. **Словарные поля (`voc_*`) — человеческим названием,** точно как в
   списках ниже, символ в символ, включая дефисы и пробелы. Не id, не
   свой вариант написания. Значение вне списка → отказ записи.
5. **Никаких полей, кроме перечисленных ниже.** Незнакомый ключ → отказ.
   В частности, у скважины **нет** поля `data_name`, не добавляй его.
6. Необязательные поля можно опускать, но не выдумывать новые.

### Закрытые словари — используй ТОЛЬКО эти значения

```

  Мамуринская, Приобская, Самотлорская, Ватьеганская,
  Тевлинско-Русскинская, Южно-Ягунская, Повховская

voc_enterprise (буровое предприятие):
  ООО "Интер-Ойл", ООО "Бурнефтесервис", АО "Сибирская буровая компания",
  ООО "ГеоБурСервис", ООО "Западбурнефть"

voc_subsoiluser (недропользователь):
  ЗАО "Самара-Нафта", АО "Приобскнефть", ПАО "Севернефтегаз",
  ООО "Ватьеганнефть"

voc_usc (тип УСЦ):
  МСЦ-146, МСЦ-168, МСЦ-178, УСЦ-140, МСЦ-219

  Назаров, Ковалёв, Ерохин, Шаймиев, Дорошенко, Бельский

voc_material (марка тампонажного материала):
  ОТМ-3, ОТМ-5, РТМ-75, РТМ-75 ПВ, ПЦТ I-G-CC-1,
  ПЦТ III-Об 5-100, ОТМ-7, РТМ-100, ЦТРО-1

  МБП-С, МБП-М, НТФ, МБП-СМ, Буфер водный, КССБ

  ПЦТ-G-CC-1, КМЦ, Бентонит, НТФ, CFL-110,
```

Кавычки внутри названий предприятий — обычные двойные, в JSON их надо
экранировать: `"ООО \"Интер-Ойл\""`.

### Поля и диапазоны

#### well — скважина

| поле | смысл | тип и диапазон |
|---|---|---|
| `int_number` | № скважины | целое, твоя квота |
| `date_done` | дата работ | `ГГГГ-ММ-ДД`, 2019-01-01 … 2025-12-31 |
| `voc_mr` | площадь / месторождение | словарь |
| `voc_enterprise` | буровое предприятие | словарь |
| `voc_subsoiluser` | недропользователь | словарь |
| `dec_depth` | глубина скважины, м | 1800 … 4200 |
| `dec_temp` | температура, °C | 45 … 110, растёт с глубиной |
| `dec_diam_col` | диаметр колонны, мм | 140 … 178 |
| `dec_diam_prev_col` | диаметр предыдущей колонны, мм | 219 … 324, всегда больше `dec_diam_col` |
| `dec_depth_prev_col` | глубина спуска предыдущей колонны, м | 900 … 2400, меньше `dec_depth` |
| `dec_depth_usc` | глубина установки УСЦ, м | 55–80 % от `dec_depth` |
| `voc_usc` | тип УСЦ | словарь; номер в названии примерно соответствует `dec_diam_prev_col` |
| `voc_technolog` | технолог | словарь |
| `voc_status` | состояние скважины | словарь; см. правило согласованности ниже |
| `ltext_remark` | примечание | свободный текст, 0–200 знаков; **заполнять примерно у трети скважин**, у остальных опускать |

**Согласованность состояния и данных — обязательна.**
`voc_status` не независимая случайная метка, а описание того, что в
записи есть:

* **Завершена** — полный набор: ступени, интервалы, испытания, АКЦ,
  горизонты. Так должно быть у большинства, примерно у семи скважин
  из десяти. `date_done` в прошлом.
* **В работе** — ступени и интервалы есть, но **нет ни одного
  `akc_by_material` и ни одного `akc_by_horizon`** (контроль качества
  ещё не проводился). Испытания есть. Примерно две скважины из десяти.
* **Запланирована** — только поля самой скважины и продуктивные
  горизонты; массив `cementing_stage` **отсутствует целиком**, АКЦ по
  пластам тоже нет. `date_done` в будущем относительно остальных.
  Примерно одна скважина из десяти.

Это важнее правдоподобия цифр: на этом признаке будут строиться
счётчики и проверяться, что выборка не ломается на неполных данных.

#### cementing_stage — ступень цементирования

Ступеней у скважины **1, 2 или 3**. Нумерация: **ступень 1 — самая
глубокая**, дальше к устью. Интервалы ступени 1 лежат ниже интервалов
ступени 2.

| поле | смысл | диапазон |
|---|---|---|
| `int_stage_number` | номер ступени | 1, 2, 3 |
| `dec_pressure_work` | рабочее давление, МПа | 8 … 20 |
| `dec_pressure_stop` | стоп-давление, МПа | на 3 … 5 больше рабочего |

#### cementing_interval — интервал цементирования

Интервалов в ступени **1–3**.

**Важно: `dec_depth_from` больше, чем `dec_depth_to`** — счёт идёт от
забоя вверх. Пример из реальных данных: `from 3000`, `to 2700`.
Интервалы внутри ступени не перекрываются и идут встык.

| поле | смысл | диапазон |
|---|---|---|
| `dec_depth_from` | верхняя цифра (глубже), м | в пределах `dec_depth` скважины |
| `dec_depth_to` | нижняя цифра (ближе к устью), м | меньше `dec_depth_from` на 200 … 900 |
| `voc_material` | марка материала | словарь |
| `dec_amount` | количество материала, т | 8 … 30 |
| `dec_volume_plan` | объём план, м³ | 5 … 35 |
| `dec_volume_fact` | объём факт, м³ | отличается от плана на −8 … +8 % |
| `dec_density_plan` | плотность план, г/см³ | 1.15 … 1.95, два знака |
| `dec_density_fact` | плотность факт, г/см³ | отличается от плана на ±0.03 |

Не указывай `calc_volume_deviation`.

#### cementing_buffer — буфер

Буферов у интервала **1 или 2** (два — примерно у каждого пятого
интервала, это важно для проверки размножения строк).

| поле | смысл | диапазон |
|---|---|---|
| `dec_volume` | объём, м³ | 1 … 5 |
| `voc_composition` | состав | словарь |
| `dec_weight` | вес, кг | 10 … 200 |

#### material_test — лабораторное испытание

**Ровно одно на интервал.**

| поле | смысл | диапазон |
|---|---|---|
| `dec_water_cement_ratio` | В/Ц | 0.38 … 0.90, два знака |
| `dec_spreadability` | растекаемость, мм | 200 … 260 |
| `dec_temp_static` | температура статика, °C | 50 … 90 |
| `dec_temp_dynamic` | температура динамика, °C | 45 … 70, меньше статики |
| `dec_pressure` | давление, МПа | 250 … 450 |
| `dec_thickening_time` | время загустевания, мин | 90 … 260 |
| `data_setting_start` | начало схватывания | **текст** вида `"6-40"`, `"2-15"` (часы-минуты) |
| `data_setting_end` | конец схватывания | текст того же вида, позже начала |
| `dec_strength_bending` | прочность на изгиб, МПа | 1.0 … 8.0 |
| `dec_strength_compress` | прочность на сжатие, МПа | 2.5 … 14.0, больше изгиба |
| `dec_linear_expansion` | линейное расширение, % | 5 … 20, **можно опускать примерно у половины** |
| `dec_water_separation` | водоотделение, % | 0.2 … 3.5 |
| `dec_fluid_loss` | водоотдача, мл/30мин | 60 … 3000 |

Обрати внимание: `data_setting_start` и `data_setting_end` — **строки
текста**, а не числа и не время. Формат ровно как в примерах.

#### akc_by_material — АКЦ по материалу

**Ровно один на интервал.** «Частичное» и «жёсткое» сцепление здесь не
две записи, а разные колонки одной записи.

| поле | смысл | диапазон |
|---|---|---|
| `dec_partial_casing_m` | частичное сцепление с колонной, м | 1 … 500 |
| `dec_partial_rock_m` | частичное сцепление с породой, м | 1 … 500 |
| `dec_partial_casing_pct` | частичное с колонной, % | 10 … 90, один знак |
| `dec_partial_rock_pct` | частичное с породой, % | 10 … 90, один знак |
| `dec_rigid_casing_m` | жёсткое сцепление с колонной, м | 20 … 600 |
| `dec_rigid_rock_m` | жёсткое сцепление с породой, м | 20 … 600 |

#### cementing_reper — репер

Висит **на ступени**, не на интервале. Есть **не у каждой ступени** —
примерно у половины; у остальных массив просто отсутствует.

| поле | смысл | диапазон |
|---|---|---|
| `dec_volume` | объём, м³ | 2 … 5 |
| `dec_density` | плотность, г/см³ | 1.30 … 1.60 |

#### cementing_reper_component — компонент репера

Компонентов у репера **2–4**.

| поле | смысл | диапазон |
|---|---|---|
| `voc_component` | компонент | словарь |
| `dec_amount` | количество | 0.01 … 2.0, два знака; **у первого компонента опускать** (основа без дозировки) |

#### productive_horizon — продуктивный горизонт

Горизонтов у скважины **2–5**.

| поле | смысл | диапазон |
|---|---|---|
| `data_formation` | пласт | текст вида `АС11`, `АС12(2)`, `БС8`, `ЮС1(1)` |
| `dec_depth_from` | кровля, м | больше `dec_depth_to`, в пределах глубины скважины |
| `dec_depth_to` | подошва, м | меньше `dec_depth_from` на 10 … 60 |
| `voc_fluid_type` | тип флюида | словарь |
| `bul_peretok` | признак перетока | `"1"` или `"0"`; **`"1"` примерно у 15 % горизонтов** |

#### akc_by_horizon — АКЦ по пласту

**Ровно один на горизонт.** Поля и диапазоны те же, что у
`akc_by_material`.

### Чего ждём от разброса

Данные нужны, чтобы на них строить отчёты и выборки. Поэтому важнее
правдоподобия — **разнообразие**:

* технологи, месторождения и предприятия должны встречаться по-разному,
  а не поровну: пусть у одного технолога будет пять скважин, у другого
  одна;
* марки материалов — с явными любимцами и редкими;
* факт должен то превышать план, то отставать, а не колебаться
  симметрично;
* `bul_peretok = "1"` должен группироваться — пусть у одной-двух скважин
  переток встретится на нескольких горизонтах сразу, а у большинства не
  встретится вовсе;
* количество ступеней, интервалов и горизонтов должно различаться от
  скважины к скважине, а не быть одинаковым.

Внутренняя согласованность обязательна: температура растёт с глубиной,
интервалы не перекрываются, стоп-давление выше рабочего, факт близок к
плану. Данные должны выдерживать взгляд человека, знающего предмет.

### Самопроверка перед ответом

Пройди по списку и убедись:

1. Ответ начинается с `{` и заканчивается `}`, ограждения нет.
2. Все значения — строки в кавычках, дробные с точкой.
3. Ни одного поля `id`, `dep_*`, `rel_main`, `active`, `data_name`
   у скважины, `calc_volume_deviation` у интервала.
4. Каждое `voc_*` значение дословно есть в списках словарей.
4a. Состояние согласовано с наполнением: у «Запланирована» нет ступеней
   вовсе, у «В работе» нет ни одной записи АКЦ, у «Завершена» есть всё.
5. `dec_depth_from` > `dec_depth_to` во всех интервалах и горизонтах.
6. `int_number` в пределах твоей квоты, без повторов.
7. У каждого интервала ровно один `material_test` и один
   `akc_by_material`.
8. У каждого горизонта ровно один `akc_by_horizon`.
9. `data_setting_start` и `data_setting_end` — текст вида `"6-40"`.
10. JSON синтаксически валиден: проверь скобки и запятые.

## Что делает Влад после получения партий

```sh
cd /var/www/html/gpdp
php tools_bulk_import.php current/testdata_dictionaries.json --dry-run
php tools_bulk_import.php current/testdata_dictionaries.json
php tools_bulk_import.php <партия>.json --dry-run
php tools_bulk_import.php <партия>.json
```

Отчёт загрузчика построчный и до конца файла — все проблемы партии видны
за один прогон, а не по одной.
