# current/

Файлы на одну операцию — не боевой код, не постоянная документация.
Сюда: разовые bootstrap-скрипты (`bootstrap_postgres.sql`), файлы
оптовой загрузки (`import_*.json`), активные `HANDOFF_*.md`-тикеты
(формат — `AI_CONTEXT_PROTOCOL.md` §4).

Отработанный `HANDOFF_*.md` — выполненное поручение, не архив: как
только целевые AI-документы пересобраны на коммит не раньше него,
файл убирается из `main` (история остаётся в git, повторно доставать
незачем). Data-файлы (`bootstrap_*`/`import_*`) не удаляются
автоматически — держим как готовый рецепт восстановления, если
понадобится пересобрать тестовую БД с нуля.

Не сюда: `docs/` — долгоживущие архивные заметки (философия,
стресс-тесты), не одноразовые операции.
