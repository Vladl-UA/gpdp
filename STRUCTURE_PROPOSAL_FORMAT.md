# Формат предложения структуры (для генерации новой модели данных)

Справочный документ (журнал 2026-07-15), пара к `BULK_IMPORT_FORMAT.md`.
Разница между ними принципиальная, не путать:

- **Этот документ** — как описать таблицы/поля/связи, которых **ещё
  нет** в базе, чтобы человек (через конфигуратор) реально их создал.
- **`BULK_IMPORT_FORMAT.md`** — как описать данные для таблиц, которые
  **уже существуют**, для загрузки скриптом `tools_bulk_import.php`.

Предложение структуры **не создаёт ничего само по себе** — это только
входные данные для решения человека. Живая база создаётся только через
конфигуратор (браузер) — у внешнего ИИ нет доступа к живой БД, и не
должно быть: создание структуры — не механическая работа, каждое поле
проходит через классификацию ниже, прежде чем стать колонкой.

## Доступные сущности (полный список, не выдумывать новые)

| entity | параметры создания | назначение |
|---|---|---|
| `data` | `name`, `label`, `length` | короткий текст (VARCHAR), `length` — максимум символов |
| `ltext` | `name`, `label` | длинный текст, без ограничения длины |
| `footnote` | `name`, `label` | текст-примечание, тот же смысл, что ltext, другая роль в форме |
| `int` | `name`, `label` | целое число |
| `dec` | `name`, `label` | дробное число, точное (DECIMAL, не FLOAT) |
| `date` | `name`, `label` | календарная дата |
| `year` | `name`, `label` | год отдельно, не часть даты |
| `time` | `name`, `label` | время суток |
| `bul` | `name`, `label` | да/нет, чекбокс |
| `voc` | `name`, `label` | выбор из словаря — по умолчанию ищет словарь `voc_<name>` |
| `link` | `name`, `label`, `target_table` | выбор из ЛЮБОЙ существующей таблицы (не только словаря), адрес явный |
| `calc` | `name`, `label`, `formula` | вычисляемое поле, формула ниже |

Структурные поля (`id`, `dep_*`, `rel_main`, `active`) — **не
перечисляются в полях таблицы**, задаются отдельно через `kind`/
`parent` самой таблицы (ниже).

## Формула для `calc`

Строго: `{имя_поля} оператор {имя_поля} оператор {имя_поля} ...`

Только `+ - * /`, строго слева направо, **без скобок и приоритета
операций** (`{a} + {b} * {c}` считается как `((a+b)*c)`, не как
математический приоритет). Имена полей — из этой же таблицы, без
префикса таблицы. Пример: `"formula": "{dec_amount} * {dec_density}"`.

## Три вида таблиц (`kind`)

- **`plain`** — обычная предметная таблица, ничем не ограничена.
- **`dictionary`** — простой словарь-склад: только `id` + `data_name`
  (подпись). Значения задаются отдельно, в `dictionary_values` (см.
  ниже), не как обычные записи.
- **`dependent`** — зависимая таблица, требует `parent`: одна
  таблица-родитель (дерево `dep_`, не сеть — у таблицы может быть
  только один родитель).

## Главное правило классификации — не путать словарь и контекст

Из `О_словарях.md`, дословно: **«Словарь хранит идентичность элемента.
Контекстная таблица хранит роль элемента и параметры его применения.»**

Если у сущности, кроме собственного названия, есть **параметры
применения** (доля, порядок, роль, статус в конкретном контексте) —
это `dependent`-таблица (контекст), не `voc_`/`dictionary`. Пример
неправильной классификации: «материал → реагент, процент участия» —
это НЕ словарь «реагент с процентом», это контекстная таблица состава
материала, где реагент — обычная ссылка (`voc_`/`link_`), а «процент»
— обычное `dec_`-поле этой контекстной строки. Не заводить `m2m`/
`junction`-таблицу с полезной нагрузкой по умолчанию — почти всегда
годится обычная зависимая таблица с ролью.

## Формат предложения (JSON)

```json
{
  "tables": [
    {
      "name": "имя_таблицы",
      "kind": "plain",
      "parent": null,
      "short": "Короткая подпись",
      "full": "Полная подпись",
      "fields": [
        {"entity": "data", "name": "title", "short": "Название", "full": "Название", "length": 255},
        {"entity": "voc",  "name": "status", "short": "Статус", "full": "Статус записи"},
        {"entity": "link", "name": "owner", "short": "Владелец", "full": "Владелец записи", "target": "person"},
        {"entity": "dec",  "name": "amount", "short": "Кол-во", "full": "Количество"},
        {"entity": "calc", "name": "total", "short": "Итого", "full": "Итоговая сумма", "formula": "{dec_amount} * {dec_price}"}
      ]
    },
    {
      "name": "имя_дочерней_таблицы",
      "kind": "dependent",
      "parent": "имя_таблицы",
      "short": "...", "full": "...",
      "fields": [ ... ]
    }
  ],
  "dictionaries": [
    {"name": "voc_status", "short": "Статус", "full": "Статус записи",
     "values": ["Черновик", "Опубликовано", "Архив"]}
  ]
}
```

## Что явно проверить перед тем, как отдавать человеку

- Каждый `voc_`/`link_` целится либо на новый словарь из `dictionaries`,
  либо на таблицу, уже описанную в этом же файле, либо на реально
  существующую (уточнить у человека, не выдумывать имя).
- `dep_`-дерево без циклов, `parent` всегда объявлен раньше по списку
  `tables`, чем ребёнок (порядок создания важен — родитель раньше).
- Имена (`name`) — латиница, цифры, `_`, с буквы (§1, идентификаторы).
- Не более одной `dependent`-связи на таблицу (одно дерево, не сеть).

## Пример — простая, некомплексная модель для затравки

Домашняя библиотека: книги, у каждой автор (словарь) и жанр (словарь),
плюс дочерняя таблица «прочтения» (когда, кем, оценка):

```json
{
  "tables": [
    {"name": "book", "kind": "plain", "parent": null,
     "short": "Книга", "full": "Книга",
     "fields": [
       {"entity": "data", "name": "title", "short": "Название", "full": "Название книги", "length": 255},
       {"entity": "voc", "name": "author", "short": "Автор", "full": "Автор"},
       {"entity": "voc", "name": "genre", "short": "Жанр", "full": "Жанр"},
       {"entity": "year", "name": "published", "short": "Год", "full": "Год издания"}
     ]},
    {"name": "book_reading", "kind": "dependent", "parent": "book",
     "short": "Прочтение", "full": "Прочтение книги",
     "fields": [
       {"entity": "date", "name": "finished", "short": "Дата", "full": "Дата прочтения"},
       {"entity": "int", "name": "rating", "short": "Оценка", "full": "Оценка (1-10)"}
     ]}
  ],
  "dictionaries": [
    {"name": "voc_author", "short": "Автор", "full": "Автор книги",
     "values": ["Булгаков", "Толстой", "Стругацкие"]},
    {"name": "voc_genre", "short": "Жанр", "full": "Жанр книги",
     "values": ["Фантастика", "Классика", "Детектив"]}
  ]
}
```
